# 打字测试系统用户旅程

本文描述当前业务模型下的两类核心用户旅程：学生旅程与管理员旅程。

## 1. 业务前提

- 学生身份主键为学号。
- 学生登录表单保留：姓名、学号、校园邮箱。
- 校园邮箱由学号自动推导，默认规则为：`{studentNo}@ucass.edu.cn`。
- 学号结构当前按以下规则解析：
  - 前 4 位：入学年份
  - 中间 5 位：专业代码
  - 中间 5 位的前 3 位：学院代码
  - 最后 2 位：班级/序号标识
- 不再维护“导入学生名单”作为系统前置条件。
- 学生登录成功后，系统自动沉淀学生档案。
- 学生端存在两条主要路径：练习、考试。

## 2. 学生用户旅程

### 2.1 首次进入系统

1. 学生打开系统首页。
2. 页面展示学生登录表单：姓名、学号、校园邮箱。
3. 学生输入学号后：
   - 系统自动校验学号格式；
   - 系统自动填充校园邮箱；
   - 系统可提示解析结果，例如入学年份、学院/专业信息（可选）。
4. 学生确认姓名、学号、邮箱无误。

### 2.2 身份认证

1. 学生点击“发送登录链接”或“发送验证码”。
2. 系统向学号对应的校园邮箱发送认证消息。
3. 学生前往校园邮箱查看邮件。
4. 学生通过 magic link 或验证码完成登录。
5. 系统登录成功后：
   - 自动创建或更新学生记录；
   - 写入学生 session；
   - 跳转到学生首页。

> 注：认证载体是 magic link 还是验证码，当前仍待和老师确认。

### 2.3 学生首页

登录成功后，学生进入统一学生首页，而不是直接开始正式考试。

首页应展示：

- 学生姓名、学号、校园邮箱
- 练习入口
- 考试入口
- 最近练习记录
- 最近考试记录
- 个人最佳成绩摘要

### 2.4 练习旅程

1. 学生点击“开始练习”。
2. 进入打字界面，系统创建一条 `practice` 模式的记录。
3. 学生完成打字并提交。
4. 系统展示本次练习成绩。
5. 学生可选择：
   - 再练一次
   - 返回学生首页
   - 查看练习历史

练习模式的体验目标：

- 低门槛
- 无需管理员配合
- 可重复进行
- 不计入正式考试排名

### 2.5 考试旅程

1. 学生点击“开始考试”。
2. 页面展示考试说明和注意事项。
3. 学生确认后进入正式考试界面，系统创建一条 `exam` 模式的记录。
4. 学生完成打字并提交。
5. 系统展示正式考试成绩。
6. 正式考试成绩纳入正式记录、排行榜与后台统计。

考试模式的体验目标：

- 明确正式性
- 与练习模式清晰区分
- 结果可回看、可统计

### 2.6 再次访问

1. 学生后续再次进入系统。
2. 仍通过学号和校园邮箱完成认证。
3. 已经考过的学生也可以继续进入系统练习。
4. 学生可继续查看自己的历史练习和历史考试记录。

## 3. 管理员用户旅程

### 3.1 登录后台

1. 管理员进入后台登录页。
2. 输入管理员账号和密码。
3. 登录成功后进入管理后台首页。

### 3.2 查看学生列表

1. 管理员进入学生管理页。
2. 系统展示“已经进入过系统的学生”列表。
3. 列表不再依赖人工导入的学生名单。
4. 每个学生可展示的信息包括：
   - 姓名
   - 学号
   - 校园邮箱
   - 入学年份
   - 学院
   - 专业
   - 最近活动时间
   - 练习次数
   - 考试次数
   - 正式最佳成绩

### 3.3 筛选与搜索

管理员可以进行多维筛选：

- 按学号/姓名/邮箱搜索
- 按入学年份筛选
- 按学院筛选
- 按专业筛选

典型使用场景：

- 查看某一年级学生的整体情况
- 查看某学院下各专业学生的表现
- 定位某个学生的练习和考试成绩

### 3.4 查看成绩详情

1. 管理员点击某个学生的“查看成绩”。
2. 系统弹出成绩详情。
3. 管理员可查看：
   - 全部记录
   - 练习记录
   - 考试记录
4. 每条记录展示速度、正确率、用时、提交时间、异常标记等。

### 3.5 管理学生状态

1. 管理员可将学生设置为启用/停用。
2. 被停用的学生下次登录或进入正式考试时会被限制。
3. 管理员也可重新启用该学生。

### 3.6 导出与复核

1. 管理员可按当前筛选条件导出成绩数据。
2. 导出可包含练习/考试模式区分。
3. 管理员可基于导出结果做教学复盘、成绩统计、领导汇报。

## 4. 旅程设计原则

### 学生侧原则

- 先登录，再分流到练习或考试
- 练习和考试必须体验上清晰区分
- 登录过程必须让学生明确知道“校园邮箱是认证方式”
- 已考过学生仍允许回到系统练习

### 管理员侧原则

- 不做无意义的前置名单维护
- 只管理真实进入过系统的学生
- 筛选维度以学号解析结果为主
- 后台以查看、统计、导出为核心，而不是手工录入
